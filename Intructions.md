# Step-by-Step Guide: How to Fork, Clone, and Contribute\n\nFollow these instructions to participate in the test.\n\n### 1. **Fork the Repository**\n1. Go to the repository URL (provided by your instructor).\n2. In the top-right corner of the page, click the **Fork** button.\n3. This will create a copy of the repository in your GitHub account.\n\n### 2. **Clone Your Fork**\n1. After forking, go to your GitHub account and open your **forked repository**.\n2. Click on the green **Code** button and copy the URL.\n3. Open your terminal and run the following command to clone your fork:\n   ```bash\n   git clone https://github.com/your-username/your-repository.git\n   ```\n4. Navigate into the cloned directory:\n   ```bash\n   cd your-repository\n   ```\n\n### 3. **Create a New Branch**\n1. Before making changes, create a **new branch** for your work. You can name it like `user-<your-username>-test`:\n   ```bash\n   git checkout -b user-<your-username>-test\n   ```\n\n### 4. **Make Changes**\n1. Edit files in the repository as required by the test.\n2. After completing your changes, save the files.\n\n### 5. **Commit and Push Changes**\n1. Stage your changes:\n   ```bash\n   git add .\n   ```\n2. Commit your changes:\n   ```bash\n   git commit -m \"Completed the test task\"\n   ```\n3. Push your changes to your fork:\n   ```bash\n   git push origin user-<your-username>-test\n   ```\n\n### 6. **Create a Pull Request (PR)**\n1. Go to the **original repository** where you forked from.\n2. You will see an option to **Create Pull Request** for the branch you just pushed.\n3. Click **New Pull Request**.\n4. Ensure the **base branch** is `main` and the **compare branch** is your own branch (`user-<your-username>-test`).\n5. Add a description if needed, then click **Create Pull Request**.\n\n---\n\n### **Important Notes**\n- **No need to merge with `main`**: You **do not** need to merge with the `main` branch in your fork.\n- **Always work on your own branch**: Do not push directly to `main`. Always create and work on a separate branch.\n- **Submit a pull request to the original repository**: After pushing to your own branch, open a pull request to merge your changes into `main` in the original repository.
